10.1 OSI 7层模型学习

亙壬2023-12-042024-01-08

OSI七层模型

OSI七层模型（Open System Interconnect）即开放系统互连参考模型，是由ISO（International Organization for Standardization）国际标准化组织提出的，用于计算机或通信系统间互联的标准体系。

  1. 应用层（Application Layer）：应用层为用户提供网络服务，包括文件传输、电子邮件、远程登录等各种应用。它与用户接口和应用程序之间的通信有关。
  2. 表示层（Presentation Layer）：表示层负责数据的格式转换、加密解密、压缩解压缩等功能，使不同系统能够互相理解数据。它还处理数据的语法和语义问题。
  3. 会话层（Session Layer）：会话层负责建立、管理和终止会话连接，以及数据交换的顺序和同步。它还处理数据的传输方式，如全双工或半双工。
  4. 传输层（Transport Layer）：传输层负责端到端的通信连接，提供可靠的数据传输服务。它通过分段和重组数据来确保数据的完整性和可靠性，并进行流量控制和错误检测。
  5. 网络层（Network Layer）：网络层负责在不同网络之间传输数据包，并处理寻址和路由选择。它的主要功能是确定数据的路径，以便数据能够从源节点传输到目的节点。
  6. 数据链路层（Data Link Layer）：数据链路层负责在相邻节点之间传输数据帧，并提供错误检测和纠正功能。它分为两个子层：逻辑链路控制（LLC）和介质访问控制（MAC）。
  7. 物理层（Physical Layer）：这一层负责管理数据传输的物理介质，包括电压、光信号、无线电频率等。它定义了连接设备的接口标准、传输速率和编码技术。

OSI七层模型提供了一个清晰的分层结构，有助于不同厂商设备之间的互操作性，同时也方便网络设备和软件开发人员了解网络通信的不同方面。

TCP/IP四层模型

TCP/IP四层模型是OSI七层模型的简化版，但是大致的核心内容是很类似的。

五层协议

五层协议的体系结构，是专门为介绍网络原理而设计的，与TCP/IP四层模型唯一不同的就是将网络接口层分为了数据链路层和物理层。

总的来说，这三种模型都是用于描述网络通信协议的体系结构。OSI7层模型是一种理论模型，因为OSI虽然是官方提出的，但是由于一些原因，在它被完善之前，由民间开发的TCP/IP早已投入实际使用了，所以它几乎是只存在与教科书中，并没有大范围地投入使用。TCP/IP4层模型是一种实际应用的模型。而五层协议是一种简化的模型。

OSI七层常用的协议

以下是 OSI 模型每一层常用的协议：

  1. 应用层：

    • HTTP （超文本传输协议）（HyperText Transfer Protocol）（端口号80）
      • 用于在Web浏览器和Web服务器之间传输超文本的协议。
    • HTTPS （HyperText Transfer Protocol Secure）（端口号443）
      • 是HTTP的安全版本，通过使用SSL/TLS加密来保护数据传输的安全性。
    • FTP （文件传输协议）（File Transfer Protocol）（端口号21）
      • 用于在网络上进行文件传输的标准协议。
    • SMTP （简单邮件传输协议）（Simple Mail Transfer Protocol）（端口号25）
      • 用于在计算机网络上发送电子邮件的协议。
    • DNS （域名系统）（Domain Name System）（端口号53）
      • 用于将域名解析为对应的IP地址的分布式数据库系统。
    • DHCP （动态主机配置协议）（Dynamic Host Configuration Protocol）（端口号67/68）
    • POP3 （Post Office Protocol 3）（端口号110）
      • 用于从远程服务器上获取电子邮件的协议。
    • IMAP （Internet Message Access Protocol）（端口号143）
      • 也是用于电子邮件的协议，允许用户从远程服务器上检索电子邮件。
    • SMB （Server Message Block）（端口号445）
      • 使用 TCP 协议在网络上传输数据。用于在网络上共享文件和打印机。SMB 协议是 Windows 操作系统的默认文件共享协议，但它也可以在其他操作系统上使用。
    • RDP （Remote Desktop Protocol）（端口号3389）
      • 使用 TCP 协议在网络上传输数据。用于远程访问和控制计算机。RDP 协议是 Microsoft Windows 操作系统的默认远程访问协议，但它也可以在其他操作系统上使用。

  2. 表示层：表示层也没有常用的特定协议，但它涉及数据格式、编码和加密解密等方面。

    • ASCII码，SSL和TLS提供安全通信，JPEG(联合图像专家组)和MPEG(动态图像压缩标准)用于图像和视频压缩

  3. 会话层：会话层并没有常用的特定协议，但它定义了建立、管理和终止会话连接的规则。

    • NetBIOS（网络基本输入输出系统）：用于网络通信
    • RPC（远程过程调用协议）
    • PPTP用于虚拟专用网络
    • SSH（安全外壳协议）（Secure Shell）（端口号22）
      • 用于在网络中提供加密的通信会话的协议。SSH 协议使用加密和认证机制来保护数据和通信。
      • SSH 协议可以用于远程访问计算机、传输文件和管理服务。
      • SSH 协议使用 TCP 协议作为传输层协议。它在应用层和传输层之间提供安全连接。

  4. 传输层：

    • TCP （传输控制协议）（Transmission Control Protocol）
      • 提供可靠的、面向连接的通信服务。
    • UDP （用户数据协议）（User Datagram Protocol）
      • 提供无连接的通信服务，适用于实时应用。
    • SCTP （流控制传输协议）

  5. 网络层：

    • IP （Internet协议）（Internet Protocol）

      • IP协议是网络层的核心协议，负责在网络中实现主机之间的通信和数据包的路由。
      • IP地址用于标识网络中的设备，IPv4和IPv6是两种常见的IP版本。

    • ICMP （Internet控制消息协议）（Internet Control Message Protocol）

      • ICMP是用于在IP网络上发送控制消息的协议，用于测试目的、错误报告和管理功能。
      • 它可以用于检测主机是否可达、测量往返时间（ping）以及发出错误报告。

    • ARP（Address Resolution Protocol）

      • 在TCP/IP4层模型中，ARP协议被归类为网络层协议，因为它解决的是IP地址到MAC地址的映射问题。

      • ARP协议用于将IP地址映射到MAC地址，以便在局域网中找到目标设备。

      • 当需要与另一台设备通信时，ARP协议会根据IP地址获取对应的MAC地址。

    • RIP（Routing Information Protocol）

      • RIP是一种用于在小型网络中交换路由信息的动态路由协议，帮助路由器决定数据包的最佳路径。
      • RIP使用跳数作为路由选择的度量标准，适用于较小规模的网络，但对大型网络不够适用。

    • OSPF （开发最短路径优先协议）（Open Shortest Path First）

      • OSPF是一种基于链路状态的动态路由协议，用于在复杂网络中计算最佳路径，并支持更大规模的网络。
      • OSPF通过派发链路状态信息并计算最短路径来确定路由，适合于大型企业网络或互联网服务提供商的使用。

    • IPSec （IP安全性协议）

  6. 数据链路层：

    • 以太网（Ethernet）
    • PPP（Point-to-Point Protocol）
    • HDLC（High-Level Data Link Control）
    • Ethernet（IEEE 802.3）定义了数据帧的格式和MAC地址的使用
    • PPP（点对点协议）
    • HDLC（高级数据链路控制）
    • ARP（地址解析协议）。在OSI7层模型中，ARP协议则被归类为数据链路层协议，因为它需要使用数据链路层的地址来进行通信。

  7. 物理层：

    • 常见的物理层协议包括以太网、Wi-Fi、蓝牙等。
    • Ethernet、RS-232、USB、光纤等
    • RJ45用于以太网接口，IEEE 802.11用于Wi-Fi无线网络

这些协议在计算机网络中扮演着重要的角色，负责实现不同层次的功能，并确保网络通信的顺利进行。

数据封装过程

在ISO/OSI七层模型中， 数据封装 是指在不同层之间传输数据时，数据在每一层都会被封装或者打包，以便在网络中传输。这个过程通常发生在数据从应用层经过传输层、网络层、数据链路层，最终到达物理层的过程中。

具体来说，数据封装的过程如下：

  1. 应用层（第七层）：应用层的数据首先被封装。这个过程包括将应用层数据打包成适合在网络上传输的格式，通常使用应用层协议（如HTTP、FTP等）进行封装。
  2. 表示层（第六层）：在表示层，数据可能会被加密、压缩或者转换成适合在网络上传输的格式。
  3. 会话层（第五层）：在会话层，数据可能被分割成更小的单元，以便在网络上传输。
  4. 传输层（第四层）：在传输层，数据会被封装成传输层协议的 数据段 （如TCP或UDP的数据段），并且会添加源端口和目的端口等信息。
  5. 网络层（第三层）：在网络层，数据会被封装成网络层协议的 数据包 （如IP数据包），并且会添加源IP地址和目的IP地址等信息。
  6. 数据链路层（第二层）：在数据链路层，数据会被封装成 数据帧 ，包括了MAC地址等信息。
  7. 物理层（第一层）：在物理层，数据会被转换成 比特流 （0和1的序列），以便通过物理介质传输。

在接收端，数据将按照相反的顺序进行解封装，直到最终到达应用层，完成整个数据传输的过程。
